## 1. 인증 (Auth)

### 1-1. 회원가입

```http
POST /api/auth/signup
Content-Type: application/json
```

**요청 예시**
```json
{
  "username": "testuser01",
  "password": "pass1234",
  "nickname": "테스트유저"
}
```

| 필드 | 타입 | 필수 | 제약조건 |
|------|------|------|----------|
| username | String | ✅ | 4~20자, 영문/숫자 |
| password | String | ✅ | 4~20자 |
| nickname | String | ✅ | 2~20자 |

**성공 응답 (201)**
```json
{
  "id": 1,
  "username": "testuser01",
  "nickname": "테스트유저",
  "role": "user",
  "createdAt": "2025-01-15T10:30:00",
  "point": 0
}
```

**에러**
- `409` : 이미 사용 중인 아이디입니다.

---

### 1-2. 로그인

```http
POST /api/auth/login
Content-Type: application/json
```

**요청 예시**
```json
{
  "username": "testuser01",
  "password": "pass1234"
}
```

**성공 응답 (200)**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "username": "testuser01",
    "nickname": "테스트유저",
    "role": "user",
    "createdAt": "2025-01-15T10:30:00",
    "point": 0
  }
}
```

> ⚠️ **중요**: 응답으로 받은 `token`을 이후 인증이 필요한 모든 요청의 `Authorization` 헤더에 포함시켜야 합니다.

**에러**
- `401` : 아이디 또는 비밀번호가 올바르지 않습니다.

---

### 1-3. 내 정보 조회

```http
GET /api/auth/me
Authorization: Bearer {token}
```

**에러**
- `401` : 인증이 필요합니다.

---

### 1-4. 회원정보 수정

```http
PATCH /api/auth/me
Authorization: Bearer {token}
Content-Type: application/json
```

**요청 예시**
```json
{
  "nickname": "새닉네임",
  "password": "newpassword"
}
```

---

### 1-5. 회원탈퇴

```http
DELETE /api/auth/me
Authorization: Bearer {token}
```

## 2. 상품 (Product)

### 2-1. 상품 목록 조회 + 검색

```http
GET /api/products
GET /api/products?search=무스비
GET /api/products?category=0&page=0
```

| 파라미터 | 타입 | 필수 | 기본값 | 설명 |
|----------|------|------|--------|------|
| page | Number | ❌ | 0 | 페이지 번호 (0부터 시작) |
| category | Number | ❌ | - | 0 = 안경, 1 = 선글라스 |
| color | String | ❌ | - | 색상 코드 (예: `#111111`) |
| search | String | ❌ | - | 상품명 검색어 |

**성공 응답 (200)**
```json
{
  "content": [
    {
      "id": 1,
      "title": "무스비 02",
      "category": 0,
      "color": "#111111",
      "soldOut": false,
      "images": ["https://example.com/image1.jpg"],
      "createdAt": "2025-01-15T10:00:00"
    }
  ],
  "page": 0,
  "size": 20,
  "hasNext": true
}
```

---

### 2-2. 상품 상세 조회

```http
GET /api/products/{id}
```

---

### 2-3. 카테고리별 상품 조회

```http
GET /api/products?category=0   // 안경
GET /api/products?category=1   // 선글라스
```

> 🔐 모든 장바구니 API는 로그인 토큰이 필요합니다.

### 5-1. 장바구니 조회

```http
GET /api/cart
Authorization: Bearer {token}
```

---

### 5-2. 장바구니 상품 추가

```http
POST /api/cart
Authorization: Bearer {token}
Content-Type: application/json
```

**요청 예시**
```json
{
  "productId": 1
}
```

**성공 응답 (200)**
```json
{
  "content": [
    {
      "cartId": 1,
      "product": {
        "id": 1,
        "title": "무스비 02",
        "category": 0,
        "color": "#111111",
        "soldOut": false,
        "images": ["https://example.com/image1.jpg"],
        "createdAt": "2025-01-15T10:00:00"
      },
      "addedAt": "2025-01-16T09:00:00"
    }
  ]
}
```

**에러**
- `400` : 잘못된 요청입니다.
- `401` : 인증이 필요합니다.
- `404` : 상품을 찾을 수 없습니다.
- `409` : 이미 장바구니에 담긴 상품입니다.

---

### 5-3. 장바구니 선택 삭제


**요청 예시**
```json

}
```

---

### 5-4. 장바구니 전체 삭제

```http
DELETE /api/cart
```

## 6. 주문 (Order)

> 🔐 모든 주문 API는 로그인 토큰이 필요합니다.

### 6-1. 주문 생성

```http
POST /api/order
Authorization: Bearer {token}
Content-Type: application/json
```

**요청 예시**
```json
{
  "items": [
    {
      "productId": 1,
      "quantity": 2
    }
  ],
  "address": "서울시 성북구 ...",
  "firstName": "홍",
  "lastName": "길동",
  "phone": "01012341234"
}
```

**성공 응답 (201)**
```json
{
  "orderId": 1,
  "totalPrice": 259000,
  "orderedAt": "2025-01-16T09:00:00"
}
```

**에러**
- `400` : 수량이 1 미만이거나 배송 정보가 누락되었습니다.
- `401` : 인증이 필요합니다.
- `404` : 상품을 찾을 수 없습니다.
- `409` : 품절된 상품은 주문할 수 없습니다.

---

### 6-2. 주문 목록 조회


---

### 6-3. 주문 상세 조회

```http

Authorization: Bearer {token}
```

---

### 6-4. 주문 취소


## 8. 공통 에러 코드

| 상태 코드 | 의미 | 주요 원인 |
|-----------|------|-----------|
| `400` | Bad Request | 요청 파라미터 오류, 필수값 누락 |
| `401` | Unauthorized | 토큰 없음, 만료, 유효하지 않음 |
| `403` | Forbidden | 권한 없음 (관리자 전용 API 일반 유저 접근 등) |
| `404` | Not Found | 해당 리소스가 존재하지 않음 |
| `409` | Conflict | 중복 데이터 (이미 존재하는 아이디, 이미 담긴 상품 등) |
| `500` | Server Error | 서버 내부 오류 |

## 9. 인증 토큰 사용법

### 토큰 저장 및 사용 예시 (JavaScript)

```javascript
// 1. 로그인 후 토큰 저장
const response = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'testuser01', password: 'pass1234' })
});
const { token } = await response.json();
localStorage.setItem('token', token);

// 2. 인증이 필요한 요청 시 토큰 첨부
const authResponse = await fetch('/api/cart', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${localStorage.getItem('token')}`,
    'Content-Type': 'application/json'
  }
});
```

### 토큰 만료 처리

- 토큰이 만료되면 서버는 `401 Unauthorized`를 반환합니다.
- 이 경우 로그인 페이지로 리다이렉트하거나 재로그인을 유도해야 합니다.
