## 1. register.html 체크박스 id 수정하기

### 문제 요약
5개 체크박스가 모두 `id="custom-checkbox"`로 되어 있어서 label 클릭이 제대로 동작하지 않습니다.

### 해결 방법

**핵심 원리**: `id`는 페이지에서 유일해야 하고, `<label for="...">`의 값은 해당 `<input>`의 `id`와 정확히 일치해야 합니다.

**수정 전**:
```html
<input type="checkbox" id="custom-checkbox" ... />
<label for="default-checkbox">모두 동의하기</label>

<input type="checkbox" id="custom-checkbox" ... />
<label for="checked-checkbox">이용약관에 동의합니다.</label>
```

**수정 후**:
```html
<input type="checkbox" id="agreeAll" ... />
<label for="agreeAll">모두 동의하기</label>

<input type="checkbox" id="agreeTerms" ... />
<label for="agreeTerms">이용약관에 동의합니다.(필수)</label>

<input type="checkbox" id="agreeAge" ... />
<label for="agreeAge">만 14세 이상입니다.(필수)</label>

<input type="checkbox" id="agreeNewsletter" ... />
<label for="agreeNewsletter">뉴스레터를 받고 싶습니다.(선택)</label>

<input type="checkbox" id="agreePrivacy" ... />
<label for="agreePrivacy">개인정보 수집 및 이용에 동의합니다.(선택)</label>
```

**확인 방법**: 브라우저에서 라벨 텍스트("모두 동의하기" 등)를 클릭했을 때, 옆의 체크박스가 토글되면 성공입니다.

## 2. goToMyPage.js — 기존 함수 재사용하기

### 문제 요약
`getProfileApi`와 `getToken`이 이미 있는데 직접 `fetch`와 `localStorage`를 사용하고 있습니다.

### 해결 방법

**수정 전** (`src/components/header/goToMyPage.js`):
```javascript
export async function goToMyPage() {
    const token = localStorage.getItem("token");       // getToken()을 안 씀
    if (!token) {
        window.location.href = "/src/components/login/login.html";
        return;
    }

    try {
        const response = await fetch("https://...", {   // getProfileApi를 안 씀
            method: "GET",
            headers: { "Authorization": `Bearer ${token}` }
        });
        // ...
    }
}
```

**수정 후**:
```javascript
import { getToken } from "../API/token/getToken.js";
import { getProfileApi } from "../API/profile/getProfileApi.js";

export async function goToMyPage() {
    const token = getToken();
    if (!token) {
        window.location.href = "/src/components/login/login.html";
        return;
    }

    try {
        await getProfileApi(token);
        // 프로필 조회 성공 = 유효한 토큰
        window.location.href = "/";
    } catch (error) {
        console.error("인증 확인 실패:", error);
        localStorage.removeItem("token");
        window.location.href = "/src/components/login/login.html";
    }
}
```

**왜 이렇게 바꾸나요?**

- API 주소가 바뀌면 `getProfileApi.js` **한 곳만** 수정하면 됩니다
- 토큰 저장 방식이 바뀌면 `getToken.js` **한 곳만** 수정하면 됩니다
- 이것이 **낮은 결합도**입니다: 변경이 한 곳에만 영향을 미치도록 만드는 것

## 7. 템플릿 로딩 함수 만들기

### 문제 요약
`orderCard.js`와 `cartCard.js`에서 HTML fetch → DOMParser → 데이터 바인딩 패턴이 중복됩니다.

### 해결 방법

공통 함수를 만들어서 중복을 제거합니다.

**새 파일**: `src/utils/loadTemplate.js`

```javascript
/**
 * HTML 파일을 가져와서 첫 번째 요소를 복제해 돌려줍니다.
 * @param {string} path - HTML 파일 경로 (예: '/src/components/order/orderCard.html')
 * @returns {DocumentFragment} - 복제된 템플릿
 */
const templateCache = {};

export async function loadTemplate(path) {
    // 같은 파일을 여러 번 fetch하지 않도록 캐시합니다
    if (!templateCache[path]) {
        const response = await fetch(path);
        if (!response.ok) throw new Error(`템플릿 로드 실패: ${path}`);
        templateCache[path] = await response.text();
    }

    const parser = new DOMParser();
    const doc = parser.parseFromString(templateCache[path], 'text/html');
    return doc.body.firstElementChild.cloneNode(true);
}
```

**사용 예시** — `orderCard.js` 수정:

```javascript
import { loadTemplate } from "../../utils/loadTemplate.js";

export async function orderCard(productContainer, data) {
    for (const product of data.data.items) {
        const card = await loadTemplate('/src/components/order/orderCard.html');

        const img = card.querySelector(".order-product-image");
        if (img) {
            img.src = product.imageUrl ?? 'https://storage.fullstackfamily.com/...';
            img.alt = product.name ?? '';
        }

        card.querySelector(".order-product-name").textContent = product.name;
        card.querySelector(".order-product-color").textContent = product.color ?? '';
        card.querySelector(".order-product-price").textContent = "₩" + product.price.toLocaleString();
        card.querySelector(".order-product-quantity").textContent = "수량: " + product.quantity;

        productContainer.appendChild(card);
    }
}
```

**달라진 점**:
- `fetch`, `DOMParser`, `parseFromString` 같은 반복 코드가 사라졌습니다
- 같은 HTML 파일을 여러 번 사용해도 네트워크 요청은 1번만 합니다 (캐시)
- `cartCard.js`에서도 같은 `loadTemplate`을 사용할 수 있습니다

## 8. checkPriceChange.js — UI와 API 분리하기

### 문제 요약
하나의 함수에 `confirm()` (UI)와 `fetch()` (API)가 섞여 있습니다.

### 해결 방법

**Step 1**: API 호출만 하는 함수를 분리합니다

`src/components/API/cart/acknowledgePriceApi.js`:
```javascript
export async function acknowledgePriceApi(token, cartItemId) {
    const API_URL = `https://api.fullstackfamily.com/api/gentlelion/v1/cart/${cartItemId}/acknowledge-price`;

    const response = await fetch(API_URL, {
        method: 'PATCH',
        headers: { 'Authorization': `Bearer ${token}` }
    });

    if (!response.ok) {
        throw new Error(`가격 인정 실패: ${response.status}`);
    }
}
```

**Step 2**: 기존 `checkPriceChange.js`는 UI 로직만 남기고, 적절한 위치로 이동합니다

`src/components/cart/checkPriceChange.js` (API 폴더가 아닌 cart 컴포넌트 폴더로):
```javascript
import { acknowledgePriceApi } from "../API/cart/acknowledgePriceApi.js";

export async function checkPriceChange(token, product) {
    if (product.price === product.prevPrice) return true;

    const confirmed = confirm(
        `${product.name}의 가격이 변경되었습니다.\n` +
        `담은 시점 가격: ₩${product.prevPrice.toLocaleString()} → ` +
        `현재 가격: ₩${product.price.toLocaleString()}\n\n` +
        `확인을 누르면 변경된 가격으로 업데이트됩니다.`
    );

    if (confirmed) {
        await acknowledgePriceApi(token, product.cartItemId);
        return true;
    } else {
        window.location.href = '/';
        return false;
    }
}
```

**분리 전후 비교**:

| | 수정 전 | 수정 후 |
|---|---|---|
| API 함수 | UI 코드(confirm)가 섞여 있음 | 순수 통신만 담당 |
| UI 함수 | API URL이 하드코딩됨 | API 함수를 import해서 사용 |
| 테스트 | API만 따로 테스트 불가 | API, UI 각각 테스트 가능 |

## 11. 인라인 스타일을 TailwindCSS로 교체하기

### 문제 요약
`orderSummation.js`와 `cartAccordion.js`에서 JavaScript로 인라인 스타일을 직접 설정합니다.

### 해결 방법

TailwindCSS 클래스를 미리 HTML에 설정해두고, JavaScript에서는 클래스만 토글합니다.

**HTML** (orderNavigation.html의 `.order-container`):
```html
<section class="order-container overflow-hidden max-h-0 transition-all duration-300 ...">
</section>
```

**JavaScript** (orderSummation.js):

```javascript
export function orderSummation() {
    const buttons = document.querySelectorAll(".order-summation-btn");
    const containers = document.querySelectorAll(".order-container");

    buttons.forEach((button, index) => {
        const container = containers[index];
        if (!container) return;

        button.addEventListener("click", () => {
            const isOpen = !container.classList.contains("max-h-0");

            if (isOpen) {
                container.classList.add("max-h-0");
                container.classList.remove("max-h-screen");
                button.querySelector("path")?.setAttribute("d", "M6 9L12 15L18 9");
            } else {
                container.classList.remove("max-h-0");
                container.classList.add("max-h-screen");
                button.querySelector("path")?.setAttribute("d", "M6 15L12 9L18 15");
            }
        });
    });
}
```

**`max-h-screen`이란?**: TailwindCSS에서 `max-height: 100vh`를 의미합니다. 컨텐츠 높이가 화면보다 크지 않다면 이걸로 충분합니다.

> **참고**: `scrollHeight`를 써야 하는 경우(컨텐츠 높이를 정확히 알아야 할 때)라면 `style.maxHeight`를 쓸 수밖에 없습니다. 하지만 대부분의 아코디언에서는 `max-h-0` ↔ `max-h-screen` 전환으로 충분합니다.

    }
});
