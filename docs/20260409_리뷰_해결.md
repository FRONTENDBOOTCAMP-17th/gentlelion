## 1. 치명적 버그 수정

### 1-1. 토큰 키 통일하기

**현재 문제**: 로그인은 `"token"`으로 저장, 상품 API는 `"admin_token"`으로 읽음

**해결 방법 A** - 상품 API 파일들의 토큰 키를 `"token"`으로 수정:

```javascript
// admin/pages/API/product/productListApi.js
// 수정 전
const TOKEN = localStorage.getItem("admin_token");

// 수정 후
const TOKEN = localStorage.getItem("token");
```

이렇게 수정해야 할 파일 목록:
- `admin/pages/API/product/productListApi.js` (3행)
- `admin/pages/API/product/productAdd.js` (4행)
- `admin/pages/API/product/productEditApi.js` (2행)
- `admin/pages/API/product/productDelete.js` (2행)
- `admin/pages/API/product/imageApi.js` (4행)

**더 좋은 방법 (해결 방법 B)** - 토큰 관리를 한 곳에서 하기:

```javascript
// admin/pages/API/auth.js (새 파일)
const TOKEN_KEY = "token";

export function getToken() {
  return localStorage.getItem(TOKEN_KEY);
}

export function setToken(token) {
  localStorage.setItem(TOKEN_KEY, token);
}

export function removeToken() {
  localStorage.removeItem(TOKEN_KEY);
}
```

그러면 모든 API 파일에서 이렇게 사용합니다:

```javascript
// admin/pages/API/product/productListApi.js
import { getToken } from "../auth.js";

export async function getProductList(page = 1, limit = 20) {
  const TOKEN = getToken();
  // ... 나머지 코드
}
```

**왜 이렇게 하나요?**
토큰 키 이름이 바뀌어도 `auth.js` 한 곳만 수정하면 됩니다. 이것이 **"결합도를 낮추는 것"**입니다. 각 API 파일은 토큰이 어떻게 저장되는지 몰라도 되고, `getToken()`만 호출하면 됩니다.

### 1-2. main.js의 undefined 변수 수정

**현재 문제**: `header` 변수가 선언되지 않아 조건문이 항상 실패

**해결 방법**: `header` 변수를 선언하거나, `headerEvent.js`에 이미 같은 코드가 있으므로 `main.js`의 중복 코드를 제거

```javascript

// 이 파일에서는 스타일 import만 담당합니다.
```

또는 `headerEvent.js`를 사용하지 않고 `main.js`에서 처리하려면:

```javascript
// src/main.js - header 변수를 직접 선언하는 방법
import "./style.css";

const btn = document.getElementById("menu-btn");
const line1 = document.getElementById("line1");

}
```

**주의**: 둘 중 하나만 선택하세요. 두 파일 모두에 같은 코드가 있으면 이벤트가 2번 등록됩니다.

### 2-3. 이미지 업로드를 공통 함수로 분리하기

```javascript
// admin/pages/components/common/imageUpload.js (새 파일)
import { uploadImage } from "../../API/product/imageApi.js";

export function initImageUpload({
  uploadBtnId = "imageUploadBtn",
  fileInputId = "imageInput",
  previewImgId = "imagePreview",
  placeholderId = "uploadPlaceholder",
  urlInputId = "imageUrlInput",
} = {}) {
  const uploadBtn = document.getElementById(uploadBtnId);
  const fileInput = document.getElementById(fileInputId);
  const previewImg = document.getElementById(previewImgId);
  const placeholder = document.getElementById(placeholderId);
  const imageUrlInput = document.getElementById(urlInputId);

  if (!uploadBtn || !fileInput) return;

  uploadBtn.addEventListener("click", () => fileInput.click());

  fileInput.addEventListener("change", async (e) => {
    const file = e.target.files[0];
    if (!file) return;

    try {
      placeholder.querySelector("span").textContent = "업로드 중...";
      const uploadedUrl = await uploadImage(file);

      previewImg.src = uploadedUrl;
      previewImg.classList.remove("hidden");
      placeholder.classList.add("hidden");
      imageUrlInput.value = uploadedUrl;
    } catch (error) {
      alert("이미지 업로드 실패: " + error.message);
      placeholder.querySelector("span").textContent = "업로드 실패. 다시 시도";
    }
  });
}
```

사용법:
```javascript
// admin/pages/components/product/add.js
import { initImageUpload } from "../common/imageUpload.js";

initImageUpload();  // 기본 ID를 사용하므로 옵션 불필요
```

## 3. 응집도 높이고 결합도 낮추기

### 3-1. API 공통 함수 만들기

모든 API 호출에서 반복되는 패턴을 하나로 묶습니다.

```javascript
// admin/pages/API/apiClient.js (새 파일)
import { getToken } from "./auth.js";

const BASE_URL = "https://api.fullstackfamily.com/api/gentlelion/v1";

export async function apiRequest(path, options = {}) {
  const {
    method = "GET",
    body = null,
    needsAuth = true,
  } = options;

  const headers = {
    "Content-Type": "application/json",
  };

  if (needsAuth) {
    const token = getToken();
    if (token) {
      headers.Authorization = `Bearer ${token}`;
    }
  }

  const fetchOptions = { method, headers };
  if (body) {
    fetchOptions.body = JSON.stringify(body);
  }

  const response = await fetch(`${BASE_URL}${path}`, fetchOptions);

  if (!response.ok) {
    throw new Error(`API 요청 실패 (${response.status})`);
  }

  return response.json();
}
```

이제 각 API 파일이 훨씬 짧아집니다:

```javascript
// admin/pages/API/product/productListApi.js (수정 후)
import { apiRequest } from "../apiClient.js";

export async function getProductList(page = 1, limit = 20) {
  try {
    const result = await apiRequest(`/admin/products?page=${page}&limit=${limit}`);
    return {
      products: result.data,
      pagination: {
        totalCount: result.meta.totalCount,
        totalPages: result.meta.totalPages,
        page: result.meta.page,
        limit: result.meta.limit,
      },
    };
  } catch (error) {
    console.error(error);
    return { products: [], pagination: { totalCount: 0, totalPages: 1, page: 1, limit } };
  }
}
```

**결합도가 낮아지는 이유**:
- 토큰 관리 → `auth.js`가 담당
- URL 구성 → `apiClient.js`가 담당
- 각 API 파일 → 자기 고유의 로직만 담당

하나를 바꿔도 나머지에 영향이 없습니다!

```javascript
window.adjustPoint = (type) => adjustPoint(type, data);
```

**수정 후** (addEventListener 사용):



**왜 이렇게 하나요?**

- `addEventListener`를 사용하면 이벤트와 관련 코드가 같은 파일에 있어서 추적하기 쉽습니다
- 나중에 코드가 커질수록 전역 변수 충돌 문제가 심해집니다

아래 순서대로 수정하는 것을 권장합니다:

### 1순위: 즉시 수정 (기능이 안 되는 것)
- [ ] 토큰 키 `"admin_token"` → `"token"` 통일 (5개 파일)
- [ ] `main.js`의 `header` 변수 문제 해결
- [ ] `dashboard.html`의 잘못된 `<td>` 수정

### 2순위: 코드 정리 (유지보수를 위해)
- [ ] `createColor()`를 `common/createColor.js`로 분리
- [ ] 이미지 업로드를 `common/imageUpload.js`로 분리
- [ ] 스크롤 이벤트를 `header.js` 한 곳으로 통합

### 3순위: 구조 개선 (장기적으로)
- [ ] `auth.js` + `apiClient.js`로 API 계층 정리
- [ ] `config.js`로 매직 넘버 관리
- [ ] `window` 전역 함수 → `addEventListener`로 변환
- [ ] 인라인 `style.cssText` → TailwindCSS 클래스로 변환

### 4순위: 코드 품질 (다음 스프린트에)
- [ ] `tailwind.config.js`에 프로젝트 색상 등록
- [ ] `header.css`의 `@apply` 줄이고 HTML에서 직접 클래스 사용
- [ ] `onclick` 인라인 이벤트 제거

## 응집도와 결합도 개념 정리

### 응집도(Cohesion) = "한 파일이 얼마나 한 가지 일에 집중하는가"

| 응집도 | 예시 | 설명 |
|--------|------|------|
| **높음** (좋음) | `common/createColor.js` | 색상 입력 필드만 만드는 파일 |
| **높음** (좋음) | `common/imageUpload.js` | 이미지 업로드만 처리하는 파일 |
| **낮음** (나쁨) | `product/add.js` (현재) | 이미지 + 색상 + 네비게이션 + 폼 전부 처리 |

### 결합도(Coupling) = "파일끼리 얼마나 서로 의존하는가"

| 결합도 | 예시 | 설명 |
|--------|------|------|
| **높음** (나쁨) | 모든 API 파일이 `localStorage.getItem("token")` 직접 호출 | 토큰 저장 방식이 바뀌면 모든 파일 수정 필요 |
| **낮음** (좋음) | `apiClient.js`를 통해 토큰 관리 | 토큰 방식이 바뀌어도 `auth.js`만 수정 |

**목표**: 응집도는 높이고, 결합도는 낮추면 **변경에 강한 코드**가 됩니다!
