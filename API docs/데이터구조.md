## 1. 전체 데이터 모델 개요

GentleLion은 안경/선글라스 쇼핑몰로, 핵심 데이터는 아래 7가지 엔티티로 구성됩니다.

```
User ──┬── Cart ──── Product
       ├── Order ─── OrderItem ─── Product
       ├── Review ── Product
       └── Wishlist ─ Product
```

모든 관계의 중심은 **User**와 **Product**입니다.

## 2. User (사용자)

### 저장 필드

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `id` | Number | 사용자 고유 ID | 자동 생성, PK |
| `username` | String | 로그인 아이디 | 4~20자, 영문/숫자, 중복 불가 |
| `password` | String | 비밀번호 | 4~20자, 서버에서 해시 처리 후 저장 |
| `nickname` | String | 닉네임 | 2~20자 |
| `role` | String | 권한 | `"user"` 또는 `"admin"` |
| `point` | Number | 보유 포인트 | 기본값 0 |
| `createdAt` | String | 가입 일시 | ISO 8601 형식 |

### 응답 예시

```json
{
  "id": 1,
  "username": "testuser01",
  "nickname": "테스트유저",
  "role": "user",
  "point": 0,
  "createdAt": "2025-01-15T10:30:00"
}
```

> ⚠️ `password`는 응답에 절대 포함되지 않습니다. 서버에서 bcrypt 등으로 해시 처리 후 저장됩니다.

### 인증 토큰 (JWT)

로그인 성공 시 서버는 JWT 토큰을 발급합니다. 토큰에는 사용자 ID와 role 정보가 인코딩되어 있으며, 인증이 필요한 모든 API 요청에 사용합니다.

```
토큰 구조: Header.Payload.Signature
저장 위치: 클라이언트 localStorage 또는 쿠키
사용 방법: Authorization: Bearer {token}
```

## 3. Product (상품)

### 저장 필드

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `id` | Number | 상품 고유 ID | 자동 생성, PK |
| `title` | String | 상품명 | 예: "무스비 02" |
| `category` | Number | 카테고리 코드 | 0 = 안경, 1 = 선글라스 |
| `color` | String | 색상 코드 | HEX 형식, 예: `"#111111"` |
| `price` | Number | 가격 | 원(KRW) 단위 |
| `soldOut` | Boolean | 품절 여부 | `true` = 품절 |
| `images` | Array\<String\> | 이미지 URL 목록 | 복수 이미지 지원 |
| `createdAt` | String | 등록 일시 | ISO 8601 형식 |

### 응답 예시

```json
{
  "id": 1,
  "title": "무스비 02",
  "category": 0,
  "color": "#111111",
  "price": 129500,
  "soldOut": false,
  "images": [
    "https://example.com/image1.jpg",
    "https://example.com/image2.jpg"
  ],
  "createdAt": "2025-01-15T10:00:00"
}
```

### 목록 조회 시 페이지네이션

```json
{
  "content": [ /* 상품 배열 */ ],
  "page": 0,
  "size": 20,
  "hasNext": true
}
```

- `page`: 현재 페이지 번호 (0부터 시작)
- `size`: 한 페이지당 상품 수 (기본 20개)
- `hasNext`: 다음 페이지 존재 여부

## 4. Cart (장바구니)

### 저장 필드

장바구니는 **User와 Product 간의 연결 테이블** 역할을 합니다.

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `cartId` | Number | 장바구니 항목 ID | 자동 생성, PK |
| `userId` | Number | 사용자 ID | FK → User |
| `productId` | Number | 상품 ID | FK → Product |
| `addedAt` | String | 장바구니 담은 일시 | ISO 8601 형식 |

### 응답 예시 (장바구니 조회)

```json
{
  "content": [
    {
      "cartId": 1,
      "product": {
        "id": 1,
        "title": "무스비 02",
        "category": 0,
        "color": "#111111",
        "soldOut": false,
        "images": ["https://example.com/image1.jpg"],
        "createdAt": "2025-01-15T10:00:00"
      },
      "addedAt": "2025-01-16T09:00:00"
    }
  ]
}
```

### 주요 규칙

- 같은 상품을 중복으로 담을 수 없습니다 (`409` 에러).
- 품절 상품도 장바구니에 담을 수 있으나, 주문 시점에 품절 체크가 이루어집니다.
- 로그인 사용자별로 독립된 장바구니를 가집니다.

## 5. Order (주문)

### 주문 저장 필드

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `orderId` | Number | 주문 고유 ID | 자동 생성, PK |
| `userId` | Number | 주문자 ID | FK → User |
| `address` | String | 배송지 주소 | 필수 |
| `firstName` | String | 주문자 성 | 필수 |
| `lastName` | String | 주문자 이름 | 필수 |
| `phone` | String | 주문자 연락처 | 필수 |
| `totalPrice` | Number | 총 결제 금액 | 원(KRW) 단위 |
| `status` | String | 주문 상태 | 아래 상태값 참고 |
| `orderedAt` | String | 주문 일시 | ISO 8601 형식 |

### 주문 상품 목록 (OrderItem)

| 필드 | 타입 | 설명 |
|------|------|------|
| `productId` | Number | 주문한 상품 ID |
| `quantity` | Number | 주문 수량 (1 이상) |
| `price` | Number | 주문 시점 상품 가격 |

### 주문 상태값

| 상태 | 설명 |
|------|------|
| `pending` | 주문 완료 (배송 준비 중) |
| `shipping` | 배송 중 |
| `delivered` | 배송 완료 |
| `cancelled` | 주문 취소 |

### 응답 예시 (주문 생성)

```json
{
  "orderId": 1,
  "totalPrice": 259000,
  "orderedAt": "2025-01-16T09:00:00"
}
```

## 6. Review (리뷰)

### 저장 필드

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `reviewId` | Number | 리뷰 고유 ID | 자동 생성, PK |
| `userId` | Number | 작성자 ID | FK → User |
| `productId` | Number | 상품 ID | FK → Product |
| `rating` | Number | 별점 | 1~5 |
| `content` | String | 리뷰 내용 | 텍스트 |
| `status` | String | 리뷰 상태 | 관리자가 수정 가능 |
| `createdAt` | String | 작성 일시 | ISO 8601 형식 |

### 응답 예시

```json
{
  "reviewId": 1,
  "userId": 1,
  "nickname": "테스트유저",
  "productId": 1,
  "rating": 5,
  "content": "정말 마음에 드는 안경입니다!",
  "createdAt": "2025-01-20T14:00:00"
}
```

## 7. Wishlist (위시리스트)

### 저장 필드

위시리스트도 **User와 Product 간의 연결 테이블** 역할을 합니다.

| 필드 | 타입 | 설명 | 비고 |
|------|------|------|------|
| `wishlistId` | Number | 위시리스트 항목 ID | 자동 생성, PK |
| `userId` | Number | 사용자 ID | FK → User |
| `productId` | Number | 상품 ID | FK → Product |
| `addedAt` | String | 찜한 일시 | ISO 8601 형식 |

### 주요 규칙

- 동일 상품을 중복으로 찜할 수 없습니다.
- 로그인 사용자별로 독립된 위시리스트를 가집니다.

## 8. 데이터 간 관계도

```
┌─────────────────────────────────────────────────────────┐
│                        USER                             │
│  id | username | password | nickname | role | point     │
└─────────────┬───────────────────────────────────────────┘
              │
      ┌───────┼──────────────────────────────────┐
      │       │                                  │
      ▼       ▼                                  ▼
  ┌───────┐ ┌─────────┐                    ┌──────────┐
  │ CART  │ │  ORDER  │                    │ WISHLIST │
  │cartId │ │orderId  │                    │wishlistId│
  │userId │ │userId   │                    │userId    │
  │prodId │ │address  │                    │productId │
  │addedAt│ │totalPrc │                    │addedAt   │
  └───┬───┘ │status   │                    └────┬─────┘
      │     └────┬────┘                         │
      │          │ (1:N)                         │
      │     ┌────▼────────┐                     │
      │     │ ORDER_ITEM  │                     │
      │     │productId    │                     │
      │     │quantity     │                     │
      │     │price        │                     │
      │     └────┬────────┘                     │
      │          │                              │
      └──────────┼──────────────────────────────┘
                 │
                 ▼
        ┌────────────────────────────────────┐
        │              PRODUCT               │
        │  id | title | category | color     │
        │  price | soldOut | images          │
        └────────────────────────────────────┘
                 ▲
                 │
        ┌────────┴───────────┐
        │       REVIEW       │
        │ reviewId | userId  │
        │ productId | rating │
        │ content | status   │
        └────────────────────┘
```

## 9. 카테고리 코드표

### 상품 카테고리 (`category`)

| 코드 | 분류 |
|------|------|
| `0` | 안경 |
| `1` | 선글라스 |

### 사용자 역할 (`role`)

| 값 | 설명 |
|----|------|
| `"user"` | 일반 사용자 (기본값) |
| `"admin"` | 관리자 (모든 기능 접근 가능) |

### 날짜/시간 형식

모든 날짜와 시간은 **ISO 8601** 형식을 따릅니다.

```
"2025-01-15T10:30:00"
 ─────────── ────────
    날짜       시간
```
