## 3-1. 헤더 HTML이 이제 6곳 이상에서 복사되고 있어요 (중요도: 높음)

### 문제

`index.html`, `sunglasses.html`, `order.html`, `header.html`, `login.html`, `register.html` 총 6곳에 거의 동일한 헤더 HTML이 복사되어 있어요.

### 해결 방법

**Step 1.** `src/components/header/loadHeader.js` 파일을 새로 만드세요.

```javascript
// src/components/header/loadHeader.js (새 파일)
export async function loadHeader(options = {}) {
  const { theme = "dark" } = options;
  // dark: 메인 페이지 (흰 글씨), light: 서브 페이지 (검정 글씨)

  const response = await fetch("/src/components/header/header.html");
  const html = await response.text();

  const parser = new DOMParser();
  const doc = parser.parseFromString(html, "text/html");
  const headerElement = doc.querySelector(".header");

  if (theme === "light") {
    const utilityBar = headerElement.querySelector(".utility-bar");
    utilityBar.classList.add("is-login-page");
  }

  document.body.insertBefore(headerElement, document.body.firstChild);
}
```

**Step 2.** 각 페이지에서 복사된 헤더 HTML을 지우고 스크립트로 교체하세요.

```html
<!-- 예: login.html — 기존 헤더 HTML 전체를 삭제하고 -->
<script type="module">
  import { loadHeader } from './loadHeader.js';
  loadHeader({ theme: 'light' });
</script>
```

> **쉽게 말하면:** 같은 편지를 6번 손으로 쓰는 대신, 원본 하나를 만들고 복사기로 찍어내는 거예요. 원본만 고치면 모든 페이지가 한 번에 바뀌어요!

## 3-2. 로그인/회원가입 폼에 `<label>`이 없어요 (중요도: 높음)

### 문제

`login.html` 392번째 줄의 이메일 입력란, `register.html`의 모든 입력란에 `<label>`이 없어요.

### 해결 방법

**Step 1.** `login.html`의 이메일 입력란에 label을 추가하세요.

```html
<!-- 수정 전 (login.html 391번째 줄) -->
<div class="relative">
  <input
    type="email"
    id="emailInput"
    placeholder="이메일 주소*"
    required
    class="w-full h-13 px-3 bg-white border border-white rounded-lg text-[13px] outline-none focus:border-black transition-all placeholder-[#858585]"
  />
</div>

<!-- 수정 후 -->
<div class="relative">
  <label for="emailInput" class="sr-only">이메일 주소</label>
  <input
    type="email"
    id="emailInput"
    placeholder="이메일 주소*"
    required
    class="w-full h-13 px-3 bg-white border border-white rounded-lg text-[13px] outline-none focus:border-black transition-all placeholder-[#858585]"
  />
</div>
```

**Step 2.** `register.html`에도 같은 방식으로 모든 입력란에 label을 추가하세요. input에 id를 먼저 추가해야 해요 (3-5번 참고).

```html
<label for="firstName" class="sr-only">이름</label>
<input id="firstName" type="text" placeholder="이름 (이름)*" required class="..." />

<label for="lastName" class="sr-only">성</label>
<input id="lastName" type="text" placeholder="성 (성)*" required class="..." />

<label for="registerEmail" class="sr-only">이메일 주소</label>
<input id="registerEmail" type="email" placeholder="이메일 주소*" required class="..." />

<label for="registerPassword" class="sr-only">비밀번호</label>
<input id="registerPassword" type="password" placeholder="비밀번호*" minlength="8" required class="..." />

<label for="confirmPassword" class="sr-only">비밀번호 확인</label>
<input id="confirmPassword" type="password" placeholder="비밀번호 확인*" minlength="8" required class="..." />
```

`sr-only`는 Tailwind CSS에서 "화면에는 안 보이지만 스크린 리더에는 읽히는" 클래스예요.

> **쉽게 말하면:** 눈을 감고 키보드만으로 회원가입한다고 상상해보세요. `<label>`이 있어야 "지금 뭘 입력해야 하는지" 음성으로 알려줄 수 있어요.

## 3-3. 로그인 폼에 JavaScript 처리가 없어요 (중요도: 높음)

### 문제

`login.html`의 `<form id="loginForm">`에 submit 이벤트 핸들러가 없어서, "계속하기" 버튼을 누르면 페이지가 새로고침될 뿐 아무 동작도 안 해요.

### 해결 방법

**Step 1.** `login.html`의 `</body>` 바로 위에 스크립트를 추가하세요.

```html
<!-- login.html — </body> 위에 추가 -->
<script type="module">
  const loginForm = document.getElementById("loginForm");
  const emailInput = document.getElementById("emailInput");

  loginForm.addEventListener("submit", (e) => {
    e.preventDefault();

    const email = emailInput.value.trim();

    if (!email) {
      alert("이메일 주소를 입력해주세요.");
      return;
    }

    // TODO: API 연동 시 아래 주석을 실제 코드로 교체
    // const response = await fetch('/api/auth/check-email', {
    //   method: 'POST',
    //   headers: { 'Content-Type': 'application/json' },
    //   body: JSON.stringify({ email })
    // });

    console.log("로그인 시도:", email);
    alert("이메일이 확인되었습니다: " + email);
  });
</script>
```

`e.preventDefault()`가 핵심이에요 — 이걸 호출하면 브라우저의 기본 폼 제출(페이지 새로고침)을 막을 수 있어요.

> **쉽게 말하면:** 편지를 쓰고 "보내기"를 눌렀는데 편지가 어디로 가야 하는지 안 정해놓은 것과 같아요. `addEventListener("submit")`이 "이 편지는 여기로 보내!"라고 정해주는 거예요.

## 3-4. `headerEvent.js`에 null 체크가 없어요 (중요도: 중간)

### 문제

`src/components/header/headerEvent.js` 15번째 줄에서 `btn`이 null일 때 19번째 줄에서 에러가 발생해요.

### 해결 방법

**Step 1.** `headerEvent.js`의 메뉴 버튼 부분을 수정하세요.

```javascript
// 수정 전 (headerEvent.js 14~27번째 줄)
//메뉴 버튼 이벤트
const btn = document.getElementById("menu-btn");
const line1 = document.getElementById("line1");
const line2 = document.getElementById("line2");

btn.addEventListener("click", () => {
  line1.classList.toggle("translate-y-[2.5px]");
  line1.classList.toggle("rotate-45");

  line2.classList.toggle("-translate-y-[2.5px]");
  line2.classList.toggle("-rotate-45");

  header.classList.toggle("is-active");
});

// 수정 후
//메뉴 버튼 이벤트
const btn = document.getElementById("menu-btn");

if (btn) {
  const line1 = document.getElementById("line1");
  const line2 = document.getElementById("line2");

  btn.addEventListener("click", () => {
    line1.classList.toggle("translate-y-[2.5px]");
    line1.classList.toggle("rotate-45");

    line2.classList.toggle("-translate-y-[2.5px]");
    line2.classList.toggle("-rotate-45");

    header.classList.toggle("is-active");
  });
}
```

> **쉽게 말하면:** "친구가 있으면 같이 놀고, 없으면 그냥 넘어가자"라는 안전장치예요. `if (btn)`이 "버튼이 있을 때만 실행해"라는 뜻이에요.

## 3-5. 회원가입 페이지의 `<input>`에 `id`가 없어요 (중요도: 중간)

### 문제

`register.html`의 모든 `<input>` 요소에 `id` 속성이 없어서 JavaScript에서 접근하기 어렵고, `<label>`과 연결할 수도 없어요.

### 해결 방법

**Step 1.** 각 `<input>`에 의미 있는 `id`를 추가하세요.

```html
<!-- 수정 전 (register.html) -->
<input type="text" placeholder="이름 (이름)*" required class="..." />

<!-- 수정 후 -->
<input id="firstName" type="text" placeholder="이름 (이름)*" required class="..." />
```

모든 입력란에 대해:

| placeholder | 추가할 id |
|---|---|
| 이름 (이름)* | `firstName` |
| 성 (성)* | `lastName` |
| 이메일 주소* | `registerEmail` |
| 비밀번호* | `registerPassword` |
| 비밀번호 확인* | `confirmPassword` |

> **쉽게 말하면:** 이름표 없는 사물함에서 물건 찾기 vs 번호 붙은 사물함에서 찾기. `id`는 "이 입력란은 3번이야!"라고 번호를 붙여주는 거예요.

## 3-6. 로그인 페이지의 SVG 아이콘 색상이 일부 맞지 않아요 (중요도: 낮음)

### 문제

로그인/회원가입 페이지의 헤더 SVG 기본 fill이 `white`로 설정되어 있어서, CSS가 로드되기 전 잠깐 흰색 로고가 보일 수 있어요.

### 해결 방법

이건 지금 당장 고치지 않아도 돼요. 3-1번(헤더 컴포넌트화)을 해결할 때 자연스럽게 해결될 거예요. 만약 바로 고치고 싶다면:

**Step 1.** `login.html`과 `register.html`의 로고 SVG에서 `fill="white"`를 `fill="#111"`로 변경하세요.

```html
<!-- 수정 전 -->
<path d="M0 14.7246..." fill="white" />

<!-- 수정 후 -->
<path d="M0 14.7246..." fill="#111" />
```

> **쉽게 말하면:** 밝은 배경에 흰 글씨를 쓰면 안 보이잖아요. 배경에 맞는 색으로 미리 설정해두는 거예요.
