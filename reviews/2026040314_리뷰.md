## 2. 잘한 점

### 코드 정리와 리팩토링
- `menuAnimation.js`에서 `main.js`와 중복되던 햄버거 애니메이션 코드를 제거한 것이 아주 좋아요! 같은 동작을 두 곳에서 하면 버그의 원인이 되는데, 이걸 발견하고 정리한 건 훌륭해요.
- `header` 변수도 `document.querySelector(".header")`로 올바르게 선언해서 이전 리뷰의 에러를 해결했어요.
- 메뉴 버튼 id를 `menu-btn`으로 통일한 것도 좋아요. 일관된 네이밍은 유지보수할 때 정말 중요해요.

### 로그인/회원가입 페이지 구현
- 로그인 페이지의 폼 디자인이 깔끔해요. `space-y-6`으로 균일한 간격, `rounded-lg`로 통일된 둥근 모서리, `transition-colors`로 부드러운 호버 효과까지 잘 적용했어요.
- 회원가입 페이지의 비밀번호 표시/숨기기 기능(`eyeIcon.js`)이 잘 구현되었어요! `querySelectorAll`로 여러 토글 버튼을 한 번에 처리하고, `DOMContentLoaded` 이벤트로 안전하게 초기화한 것이 좋아요.
- 비밀번호 입력 필드에 `minlength="8"` 브라우저 기본 유효성 검사를 활용한 것도 좋은 접근이에요.

### CSS 스타일 확장
- `src/style.css`에 `.is-login-page` 클래스를 추가해서 로그인 페이지 전용 헤더 스타일을 만든 것이 스마트해요. 기존 `is-scrolled`, `is-active` 패턴을 그대로 따라가면서 확장한 것이 일관성 있어요.
- 로그인 페이지의 서브메뉴에서 `grid-rows-[0fr]` → `grid-rows-[1fr]` 애니메이션 기법은 최신 CSS 트릭이에요! `max-height` 방식보다 더 매끄러운 애니메이션을 만들어주는 고급 기법이에요.

### API 문서
- `docs/api.md`에 API 설계 문서를 추가한 것이 프로젝트 완성도를 높여줘요. 인증, 상품, 주문 등 체계적으로 정리되어 있어요.

### 결제 페이지
- `order.html`에서 결제 UI를 구현한 것이 좋아요. 모바일 메뉴 애니메이션과 헤더 이벤트도 잘 연동되어 있어요.

## 3. 개선하면 좋을 점

### 3-1. 헤더 HTML이 이제 6곳 이상에서 복사되고 있어요 (중요도: 높음)

**현재 상태:**

이전 리뷰에서 2곳(index.html, sunglasses.html)이었던 헤더 중복이 이제 더 늘어났어요:

1. `index.html`
2. `src/pages/sunglasses.html`
3. `src/pages/order.html`
4. `src/components/header/header.html`
5. `src/components/login/login.html`
6. `src/components/login/register.html`

**왜 개선하면 좋을까?**
페이지가 추가될 때마다 헤더를 복사-붙여넣기하고 있어요. 메뉴 항목 하나를 수정하려면 6개 파일을 모두 고쳐야 해요. 이미 `login.html`과 `register.html`에서는 드롭다운 애니메이션 방식이 다른데(`grid-rows` vs `max-height/opacity`), 이것은 동기화가 안 된 증거예요.

**추천:** `header.html`이 이미 별도 파일로 분리되어 있으니, JavaScript로 동적 삽입하는 방식을 도입하세요.

```javascript
// src/components/header/loadHeader.js
async function loadHeader() {
  const response = await fetch('/src/components/header/header.html');
  const html = await response.text();
  const parser = new DOMParser();
  const doc = parser.parseFromString(html, 'text/html');
  const headerElement = doc.querySelector('.header');
  document.body.insertBefore(headerElement, document.body.firstChild);
}

loadHeader();
```

### 3-2. 로그인/회원가입 폼에 `<label>`이 없어요 (중요도: 높음)

**현재 상태:**

```html
<!-- login.html 392번째 줄 -->
<input
  type="email"
  id="emailInput"
  placeholder="이메일 주소*"
  required
  class="w-full h-13 px-3 ..."
/>
<!-- label이 없고 placeholder만 있어요 -->
```

```html
<!-- register.html — 비밀번호 필드도 마찬가지 -->
<input type="password" placeholder="비밀번호*" minlength="8" required ... />
<!-- id도 없고 label도 없어요 -->
```

**왜 개선하면 좋을까?**
`<label>`이 없으면 스크린 리더 사용자가 이 입력란이 뭔지 알 수 없어요. `placeholder`는 입력을 시작하면 사라지기 때문에 `label`을 대체할 수 없어요. 또한 `<label>`을 클릭하면 해당 입력란에 포커스가 되어 사용성도 좋아져요.

**추천:**
```html
<label for="emailInput" class="sr-only">이메일 주소</label>
<input
  type="email"
  id="emailInput"
  placeholder="이메일 주소*"
  required
  class="w-full h-13 px-3 ..."
/>
```

Tailwind의 `sr-only` 클래스를 쓰면 화면에는 보이지 않지만 스크린 리더에는 읽혀요.

### 3-3. 로그인 폼에 JavaScript 처리가 없어요 (중요도: 높음)

**현재 상태:**

```html
<!-- login.html 386번째 줄 -->
<form id="loginForm" class="space-y-6">
  <!-- ... 입력 필드들 ... -->
  <button type="submit" class="...">계속하기</button>
</form>
<!-- form에 대한 submit 이벤트 핸들러가 없어요 -->
```

**왜 개선하면 좋을까?**
`<form>`의 `submit` 버튼을 클릭하면 브라우저 기본 동작으로 **페이지가 새로고침**돼요. API 설계 문서(`docs/api.md`)까지 만들어 놓았으니, 폼 제출 시 API를 호출하는 JavaScript 코드가 필요해요. 없으면 이메일을 입력하고 "계속하기"를 눌러도 아무 일도 안 일어나고 페이지만 새로고침돼요.

**추천:**
```javascript
// login.html에 script 추가
const loginForm = document.getElementById("loginForm");

loginForm.addEventListener("submit", (e) => {
  e.preventDefault(); // 페이지 새로고침 방지

  const email = document.getElementById("emailInput").value;

  // 이메일 형식 확인
  if (!email) {
    alert("이메일을 입력해주세요.");
    return;
  }

  // TODO: API 호출 (docs/api.md 참고)
  console.log("로그인 시도:", email);
});
```

### 3-4. `headerEvent.js`에 null 체크가 없어요 (중요도: 중간)

**현재 상태:**

```javascript
// src/components/header/headerEvent.js 15~19번째 줄
const btn = document.getElementById("menu-btn");
const line1 = document.getElementById("line1");
const line2 = document.getElementById("line2");

btn.addEventListener("click", () => {  // btn이 null이면 에러!
```

**왜 개선하면 좋을까?**
`headerEvent.js`를 사용하는 모든 페이지에 `id="menu-btn"` 요소가 있어야 해요. 만약 데스크톱 전용 페이지처럼 메뉴 버튼이 없는 페이지에서 이 스크립트를 로드하면 에러가 발생해요.

```javascript
const btn = document.getElementById("menu-btn");

if (btn) {
  const line1 = document.getElementById("line1");
  const line2 = document.getElementById("line2");

  btn.addEventListener("click", () => {
    line1.classList.toggle("translate-y-[2.5px]");
    line1.classList.toggle("rotate-45");
    line2.classList.toggle("-translate-y-[2.5px]");
    line2.classList.toggle("-rotate-45");
    header.classList.toggle("is-active");
  });
}
```

### 3-5. 회원가입 페이지의 `<input>`에 `id`가 없어요 (중요도: 중간)

**현재 상태:**

```html
<!-- register.html — 입력 필드들에 id가 없어요 -->
<input type="text" placeholder="이름 (이름)*" required class="..." />
<input type="text" placeholder="성 (성)*" required class="..." />
<input type="email" placeholder="이메일 주소*" required class="..." />
<input type="password" placeholder="비밀번호*" minlength="8" required class="..." />
<input type="password" placeholder="비밀번호 확인*" minlength="8" required class="..." />
```

**왜 개선하면 좋을까?**
나중에 JavaScript로 폼 데이터를 가져올 때 `id`가 없으면 입력값을 읽을 방법이 제한돼요. `document.getElementById()`를 쓸 수 없고, `querySelector`로 복잡하게 찾아야 해요. `<label>`과 연결하려면 `id`가 반드시 필요해요.

**추천:**
```html
<input id="firstName" type="text" placeholder="이름 (이름)*" required class="..." />
<input id="lastName" type="text" placeholder="성 (성)*" required class="..." />
<input id="registerEmail" type="email" placeholder="이메일 주소*" required class="..." />
<input id="registerPassword" type="password" placeholder="비밀번호*" minlength="8" required class="..." />
<input id="confirmPassword" type="password" placeholder="비밀번호 확인*" minlength="8" required class="..." />
```

### 3-6. 로그인 페이지의 SVG 아이콘 색상이 일부 맞지 않아요 (중요도: 낮음)

**현재 상태:**

```html
<!-- login.html — 헤더 SVG path의 fill이 "white" -->
<path d="M0 14.7246..." fill="white" />

<!-- 하지만 CSS에서 .is-login-page로 검정색으로 덮어씌움 -->
.utility-bar.is-login-page .logo-item svg path {
  fill: #111;
}
```

배경이 `#F3F4F6`(밝은 회색)인데 SVG 기본 fill이 `white`예요. CSS가 로드되기 전 잠깐 흰색 로고가 보일 수 있어요 (FOUC — Flash of Unstyled Content).

**왜 개선하면 좋을까?**
페이지 로딩 시 로고가 잠깐 흰색으로 보였다가 검정색으로 바뀌는 깜빡임이 발생할 수 있어요.

**추천:** 로그인 페이지용 헤더에서는 SVG의 기본 fill을 `#111`로 설정하거나, CSS가 충분히 빠르게 로드되도록 `<link>` 태그 순서를 확인하세요.

## 6. 마무리

이전 리뷰에서 지적한 `menuAnimation.js` 문제를 깔끔하게 해결한 것이 인상적이에요! 중복 코드를 찾아서 제거하고, 변수 선언 누락을 수정하는 건 실제 개발에서 정말 중요한 디버깅 스킬이에요.

새로 추가된 로그인/회원가입 페이지와 결제 페이지도 UI가 잘 만들어져 있고, CSS `grid-rows` 애니메이션 같은 최신 기법까지 시도한 것이 대단해요. API 문서까지 준비한 것은 프로젝트를 체계적으로 진행하고 있다는 증거예요.

지금 가장 중요한 건 **폼에 JavaScript 로직을 연결하는 것**이에요. UI는 이미 완성했으니, 이제 "입력 → 검증 → API 호출" 흐름을 구현하면 진짜 동작하는 쇼핑몰이 되어가요. 화이팅!
